## 2. TypeScript Features Used

1. **Basic types** (`string`, `number`, `boolean`) — typed variable declarations for course properties and app metadata
2. **Arrays** (`string[]`, `number[]`) — typed arrays for course lists and scores
3. **Tuples** (`[string, number]`) — fixed-length typed array for course entry pairs
4. **Enums** (`enum StudyStatus`) — named constants for tracking study progress (NotStarted, InProgress, Completed)
5. **Type aliases** (`type CourseInfo = { ... }`) — custom object types for course data
6. **Interfaces** (`interface ITracker`) — contract defining tracker behavior
7. **Arrow functions** — used extensively for event handlers, formatting utilities, and callbacks; demonstrates single-expression bodies, multiline bodies, default parameters, and optional parameters
8. **Default parameters** — `formatTitle(title, prefix = "📚")` provides a fallback value
9. **Optional parameters** — `formatCredits(credits?: number)` handles missing values
10. **Spread operator** — `[...defaultCourses, "TypeScript", "Node.js"]` to merge arrays
11. **Object destructuring** — `const { title, description, credits } = sampleCourse`
12. **Array destructuring** — `const [firstScore, secondScore, thirdScore] = scores`
13. **Classes** — `Course` class with typed fields, constructor, and methods
14. **Inheritance** — `AdvancedCourse extends Course` adds prerequisite field and overrides getter
15. **Access modifiers** — `public`, `private` on class properties and methods
16. **Getters and setters** — `get summary()` and `set status()` in Course; `get name()` / `set name()` in CourseName with validation
17. **Static members** — `Course.totalCreated` tracks total instances across all courses
18. **Abstract classes** — `abstract class BaseTracker` with abstract `render()` method
19. **Union types and type guards** — `function describeValue(value: number | string)` with `typeof` checks
20. **Generics** — `function findItem<T>(items: T[], predicate: (item: T) => boolean)`
21. **Function types** — `let statusFormatter: (status: StudyStatus) => string`
22. **The `any` type** — demonstrated for flexible variable assignment
23. **Hex and octal number literals** — `0x0064a4`, `0o755`

## 3. Cross-Browser Testing Considerations

The TypeScript code is compiled to ES2015 JavaScript using the TypeScript compiler (`tsc`), which strips all type annotations and transpiles TypeScript-specific constructs (enums, access modifiers, etc.) into standard JavaScript. The resulting `app.js` runs in any modern browser that supports ES2015+ (Chrome, Firefox, Safari, Edge — all current versions). The HTML page uses only standard HTML5 elements and CSS3 styling with no vendor prefixes required. To verify cross-browser compatibility, open `index.html` in different browsers and confirm that the course list renders, the "Add" form works, status cycling operates correctly, and the browser console shows no errors.
