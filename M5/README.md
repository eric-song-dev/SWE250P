## 2. Advanced TypeScript Features Used

### Chapter 6 Features (in `src/`)

1. **Interfaces** (`interface ICourse`) — defines the shape of course objects with typed properties and method signatures
2. **Optional interface properties** (`grade?: string`) — allows properties that may or may not be present
3. **Interface methods** (`getLabel(): string`) — method signatures enforced by the interface contract
4. **Interface extension** (`interface IAdvancedCourse extends ICourse`) — creates a sub-interface that adds `prerequisite` and `difficulty`
5. **Class implementing interface** (`class Course implements ICourse`) — class must satisfy all interface requirements
6. **Type assertion** (`let c = {} as ICourse`) — tells the compiler to treat an object as a particular type
7. **Namespaces** (`namespace Formatter { ... }`) — groups related functions under a named scope to avoid global pollution
8. **Decorators** (`@logConstructor`) — class decorator that logs the constructor when the class is registered
9. **ES modules** (`import { ... } from "./models.js"`) — `export` and `import` for organizing code across files
10. **String enums** (`enum StudyStatus { NotStarted = "Not Started", ... }`) — enum values as human-readable strings
11. **Source maps** (`sourceMap: true` in tsconfig) — maps compiled JS back to original TS for debugging
12. **Declaration files** (`declaration: true`) — generates `.d.ts` type definition files

### Chapter 7 Features (in `webpack-demo/`)

13. **Webpack bundling** — `src/index.js` is bundled into a single optimized `dist/main.js`
14. **Minification** — Webpack production mode automatically minifies the output
15. **Webpack Configuration** — tailored via `webpack.config.js` (entry, output, mode)
16. **Dev Server** (`webpack-dev-server`) — provides live reloading and a local development server
17. **Plugins** (`TerserPlugin`) — explicitly configured for minification options

## 3. Debugging TypeScript Code

TypeScript debugging is enabled through **source maps**, which are generated by setting `"sourceMap": true` in `tsconfig.json`. When you compile TypeScript to JavaScript, the compiler produces `.js.map` files that create a mapping between the generated JavaScript and the original TypeScript source. This allows browser developer tools (such as Chrome DevTools) to display your original `.ts` files in the Sources panel, set breakpoints on TypeScript lines, and step through TypeScript code even though the browser is actually executing JavaScript. You can also insert the `debugger;` statement directly into your TypeScript code to create a programmatic breakpoint that triggers when DevTools is open. Additionally, running `tsc --watch` enables continuous compilation, which immediately reports type errors as you save files, providing a fast feedback loop during development. For Webpack projects, you can add `devtool: "source-map"` to your `webpack.config.js` to generate source maps for bundled code as well.

## 4. Cross-Browser Testing Considerations

The TypeScript code is compiled to ES2015 JavaScript with ES module output (`import`/`export`), loaded via `<script type="module">`. ES modules are supported natively in all modern browsers (Chrome 61+, Firefox 60+, Safari 11+, Edge 16+). The Webpack bundle in `webpack-demo/` is a single self-contained file that runs in any browser supporting ES5+. To test cross-browser compatibility, open both `index.html` (the module-based app) and `webpack-demo/dist/index.html` (the bundled app) in Chrome, Firefox, Safari, and Edge, and verify that the UI renders correctly, interactions work, and the console shows no errors.

## 5. How to Build and Test

```bash
# Part 1: Advanced TypeScript with Modules
cd M5
npm install
npx tsc                    # Compiles src/*.ts → dist/*.js

# Open in browser (must use an HTTP server for ES modules)
# Option A: Python
python3 -m http.server 8080
# Then visit http://localhost:8080/index.html

# Option B: npx
npx serve .

# Part 2: Webpack Demo
cd webpack-demo
npm install
npm run build              # Bundles src/index.js → dist/main.js explicitly using webpack.config.js
npm start                  # Starts webpack-dev-server on port 9000 (hot reloading enabled)
```
